# AI 视频创作平台 — UI 质量审查 & 品牌质感优化计划

**审查日期**: 2026-04-29  
**审查依据**: 花叔Design 设计哲学 + 反 AI slop 清单  
**设计师定位**: 本次 embodied as 品牌 UX 设计师，以「AI 视频工厂」的产品身份审视视觉品质

### 1.2 设计系统评估

```
品牌色板:     #7CB342 (绿)  ← 唯一的品牌色
辅助色:       #5B8DEF (Smart蓝) #7C3AED (Expert紫) ← 仅声明未使用
灰度:         #1d1d1f → #86868b → #aeaeb2 → #e8e8ed → #f5f5f7
语义色:       #ff453a (error) #30d158 (success) #ff9f0a (warning)
```

**问题**: 品牌绿是唯一被实际使用的色。辅助 accent 色（Smart 蓝 / Expert 紫）在 CSS 变量中声明了但组件中从未引用。这使整个平台在视觉上只有「绿色 + 灰色」两个维度，缺少场景切换时的视觉信号差异。

## 二、逐页逐组件审查

### 2.1 SplashScreen（启动页）

**当前**: 品牌绿全屏底 + splash-final.png 图片叠加 contain，手动计算按钮坐标映射。

**问题**:
- 启动图是外部设计的静态图，与产品 UI 风格割裂——进入后从「插图世界」跳到「iOS 控制台」，视觉连续感断裂
- 按钮坐标硬编码（`BTN_X=72 Y=820`），图片更新后即失效

**建议**: 将启动页改为品牌动画序列——Logo 渐现 → 品牌绿背景淡入 → 3 个场景关键词轮播 → 「开始创作」按钮 scaleIn。整体 2-3 秒，内外风格统一。

---

### 2.2 主页面状态机（page.tsx）

**当前**: 4 阶段状态机（home → recommend → generate → result），所有视图在同一组件中条件渲染。

**问题**:
1. **阶段切换无过渡动画** — 从表单页跳到推荐页再跳到结果页，「硬切」让用户感觉不连贯
2. **状态变量过多** — 15+ 个 useState，代码可读性差，视觉上也无法形成清晰的心智模型
3. **Smart Create / Expert Studio 双模式切换** — 仅在按钮上体现，没有视觉氛围变化来区分两种模式
4. **推荐阶段 (RecommendPanel) 缺乏「设计决策感」** — 目前是文字列表 + 卡片，没有传达「AI 在帮你思考」的智能感

**关键改进**:
- 🔴 **阶段间过渡**: 每个 stage change 加 300ms fade + slide，建立空间感
- 🟡 **Smart/Expert 双模式视觉区分**: Smart 模式用品牌绿 + 微暖色温，Expert 模式用辅助 accent 色 + 更克制的信息密度
- 🟡 **RecommendPanel 设计升级**: 从文字卡片 → 视觉 moodboard + 策略卡片并排

---

### 2.3 SceneTabs（场景选择）

**当前**: 3 个横向按钮，绿色边框高亮选中态。

**评分**: ⭐⭐⭐⭐ — 干净有效，信息层级清晰。

**微调建议**: 选中态加一个微小的绿色指示条（2px 底部 border），替代当前的 ring 效果——ring 在亮色 UI 上容易「糊」。

---

### 2.4 SceneForm（产品/场景输入表单）

**当前**: 单页长表单，分段显示（产品信息 / 品牌声音 / 高级设置），每个场景对应不同字段。

**问题**:
1. **表单过长** — Product Direct 场景下有 8+ 个输入框 + 3 个展开区，对新用户压迫感强
2. **标签和占位符拥挤** — `text-[11px]` 标签 + placeholder 内容，字号过小
3. **高级设置默认折叠** — 好设计，但折叠/展开无动画
4. **品牌色使用过少** — 表单区几乎全是灰色 + 白色，品牌绿仅出现在提交按钮上

**关键改进**:
- 🟡 **表单视觉重量重平衡**: 当前每个字段权重均等 → 按信息重要度分「核心字段」(产品名+卖点) 用 14px + 略大卡片；「辅助字段」折叠在下方
- 🟡 **品牌绿注入**: 聚焦态 input border → 品牌绿；表单分区标题加绿色左侧 accent 短线
- 🟢 **平台选择改用 Toggle 而非 checkbox 列表** — 当前是横向 pill 选择，但在 S3/S4 场景下不明显

---

### 2.5 ReviewPanel（审核面板）

**当前**: 左侧审核报告评分卡 + 右侧内容预览，底部 approve/reject/changes 三个按钮。

**问题**:
1. **审核操作按钮层级不清晰** — approve/reject/changes 三者并排等宽，视觉上没有区分主次
2. **评分卡信息密度偏高** — AuditScoreCard 展示所有 criteria，但缺少「一眼看出问题严重度」的视觉速记
3. **审核节点切换无视觉提示** — 从 strategy review → script review → edit review → thumbnail review，用户需要感知「进度」

**关键改进**:
- 🔴 **审核按钮优先级**: Approve 主按钮（品牌绿填充）→ Request Changes 次按钮（outline）→ Reject 危险按钮（红色 outline，放右侧）
- 🟡 **进度指示器**: 4 个审核点横向 steps，当前完成/等待中/正在审核 三态视觉区分

---

### 2.6 OneShotResultView（结果展示）

**当前**: 多 tab 结果页（briefs/scripts/videos/thumbnails/media/quality/performance/raw），8 个 tab。

**问题**:
1. **Tab 过多** — 8 个 tab 超过人类短期记忆容量（7±2），用户需要来回扫才能找到目标
2. **Tab 图标使用内联 SVG path** — 每个 tab 的 icon 是手写的 `<svg>` path，维护成本高、不统一
3. **Demo Mode 占位符** — 灰色虚线卡片 + "Demo Mode" 标签，虽诚实但缺乏设计温度

**关键改进**:
- 🔴 **Tab 归并**: 8 → 4 个核心 tab（内容 / 媒体 / 质量 / 原始数据），内容 tab 内嵌子导航
- 🟡 **Demo 占位符升级**: 保留诚实性，但给占位符加微妙的品牌色淡化背景 + 「连接后端后自动替换」的说明
- 🟢 **视频预览区**: 当有 final_video_path 时，视频播放器应是页面视觉重心，目前权重不足

---

### 2.7 Nav（导航栏）

**当前**: 横向链接 + 语言切换 pill。

**评分**: ⭐⭐⭐⭐ — 简洁有效。

**问题**: 导航缺少品牌标识（Logo）。一个工具型产品，左上角放产品名/Logo 是最基本的品牌识别锚点。

**建议**: 左侧加产品名「Short Video Factory」或 Logo。语言切换 pill 的 `text-[10px]` 过小。

---

### 2.8 Brand Packages / Footage / Influencers（子页面）

**当前**: CRUD 列表页，重复使用 apple-card + apple-btn 组件体系。

**评分**: ⭐⭐⭐ — 功能完整但缺乏「管理后台」的专业感。

**共同问题**:
1. 空状态设计过于朴素 — 仅灰色大图标 + 文字，没有品牌情感
2. 卡片列表缺少微妙的视觉层次 — 所有卡片视觉权重相同
3. 表单 modal 设计合格，但没有键盘快捷键支持

**建议**: 空状态加品牌绿淡色插图或一句话产品价值主张。

## 三、品牌质感综合评分

按花叔Design 5 维度评审：

| 维度 | 得分 | 评语 |
|------|------|------|
| **哲学一致性** | 7/10 | Apple Design System v3 方向明确，但执行不彻底——品牌绿仅作为 accent 使用，未形成强烈的品牌识别 |
| **视觉层级** | 6/10 | 信息架构清晰但视觉权重均等化，缺少年度/节奏。审核按钮、结果 tab 等关键操作未做视觉区分 |
| **细节执行** | 7/10 | CSS 变量体系、动画关键帧、组件一致性做到位。11px/10px 字号偏多，可读性受限 |
| **功能性** | 8/10 | 状态机完整，错误/加载/空状态全覆盖，轮询/离线检测/会话恢复完善 |
| **创新性** | 5/10 | 遵循成熟范式（Apple HIG + Tailwind），未展现 AI 视频工厂应有的「智能感」或「创作感」视觉语言 |

**总评: 6.6/10 — 良好的工具型 UI，但未达到「有记忆点的品牌产品」标准**

## 四、品牌质感升级计划

### P0 · 立即修复（1-2h，ROI 最高）

| # | 修复项 | 当前 | 目标 | 影响 |
|---|--------|------|------|------|
| P0-1 | 审核按钮层级 | 三按钮等宽 | Approve 主按钮 → Changes outline → Reject 红 outline 右置 | 减少误操作，提升审核效率 |
| P0-2 | Result Tab 归并 | 8 个 tab | 4 个主 tab（内容/媒体/质量/数据）+ 子导航 | 降低认知负荷 |
| P0-3 | Nav 加 Logo/产品名 | 无标识 | 左侧「Short Video Factory」+ 品牌绿圆点 | 建立品牌锚点 |
| P0-4 | 阶段切换加过渡动画 | 硬切 | 300ms fade+slide 过渡 | 视觉连贯感 |

### P1 · 本周完成（3-5h）

| # | 修复项 | 当前 | 目标 |
|---|--------|------|------|
| P1-1 | Smart/Expert 双模式视觉氛围 | 仅按钮文案不同 | Smart 模式暖绿氛围 + 自动化感；Expert 模式克制深色 + 手动控制感 |
| P1-2 | 审核进度指示器 | 无 | 4 步横向 steps（strategy→script→edit→thumbnail），三态区分 |
| P1-3 | 表单视觉重量重平衡 | 所有字段权重均等 | 核心字段 14px + 略大卡片；辅助字段折叠 |
| P1-4 | 启动页品牌动画 | 静态图片 | 2-3s 品牌动画序列（Logo → 场景轮播 → CTA） |
| P1-5 | 空状态品牌化 | 灰色图标 | 品牌绿淡色插图 + 产品价值主张一句话 |

### P2 · 迭代优化（持续）

| # | 修复项 | 说明 |
|---|--------|------|
| P2-1 | 暗色模式 | 视频创作者常夜间工作，暗色模式是必需品不是奢侈品 |
| P2-2 | 配色扩展 | 为 4 个场景各分配一个辅助色（不偏离品牌绿主调），场景切换时有视觉信号 |
| P2-3 | 排版节奏 | 建立 4/8/12/16/24/32 的 spacing rhythm，替代散落的 px-[n] 工具类 |
| P2-4 | 「创作感」微交互 | 生成进度不仅显示文字，加 subtle 的粒子/波纹动画传达「AI 在工作」 |
| P2-5 | 键盘快捷键 | Esc 关 modal、Cmd+Enter 提交审核、1-4 切换场景 |

## 五、执行纪律

1. **不凭空发明新颜色**: 所有新增色从现有 CSS 变量体系派生
2. **每个改动配 before/after 截图**: 用 Playwright 截图对比
3. **保持组件 API 兼容**: 不改 props 签名，只改内部渲染
4. **国际化同步**: 新增文案同步添加到 zh/en 翻译表
5. **Design token 先于组件**: 新增颜色/字号/间距先在 `:root` 定义 CSS 变量，再引用
