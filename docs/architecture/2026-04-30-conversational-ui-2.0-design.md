## 一、当前问题 vs 设计目标

| 现状痛点 | 设计目标 |
|---------|---------|
| 密密麻麻表单 | 矩阵导航：横轴选场景，纵轴选视频类型，卡片层展开 |
| 无引导力 | 每张卡片：引导问题 → 原因说明 → 输入框 |
| 没有方法论 | 卡片间连接线 + 文字说明展示上下游关系 |
| 执行页是黑盒 | 叙事化进度 + 非阻塞状态条 |
| 结果页信息爆炸 | 导演回放滚动叙事 |
| 发布割裂 | 结果页底部自然衔接，一键发布 |
| 数据看不懂 | 按视频类型的成绩单 + AI 白话总结 |
| 作品集像仓库 | 创作画廊双轴浏览 |

## 二、产品页面架构

```
导航栏
├── 首页 (创作入口)
│   └── GuidedForm (矩阵导航 + 引导卡片流)
├── 创作画廊
│   ├── 成品 tab (按场景分组)
│   └── 素材 tab (按类型分组)
└── 品牌资产
    ├── 品牌身份 (Brief · 画像 · 规范 · 产品图 · 参考)
    └── 已产出内容 (AI视频 · AI图片 · AI音频 · 品牌原创)
```

**全链路数据流**：

```
输入页          执行页          结果页 (单页滚动)
GuidedForm  →  叙事进度  →  ┌─ 视频播放器
    │                       ├─ 创作脚本
    │                       ├─ 关键帧画廊
    │                       ├─ 质量报告
    │                       ├─ 发布区 (6.5)
    │                       ├─ 数据复盘 (6.6)
    │                       └─ → 查看创作画廊
    │
    └──→ 素材自动归档品牌资产
```

## 三、三层导航矩阵

### 3.2 纵轴 — 选中场景下的视频类型

| 场景 | 视频类型 |

### 3.3 卡片层 — 引导卡片

卡片按三层优先级从上到下瀑布展开，层间用带箭头和引导文字的连接线串联。

**卡片结构**：每张卡片固定四行——优先级标识+步骤编号 / 引导问题 / 原因说明 / 输入框。

**优先级视觉系统**：

| 层级 | 左边框 | 标签 | 默认状态 |
|------|--------|------|---------|
| 🔴 必填 | `#6A2B3A` | "必填" | 始终展开 |
| 🟡 建议填 | `#B27A7E` | "推荐填写" | 始终展开 |
| 🟢 可选 | `#6B8578` | "可选项" | 折叠，点击展开 |

**步骤隐喻名**：

| 字段 | 步骤名 | 图标 |
|------|--------|------|
| 产品名称 | 定主角 | 🎯 |
| 核心卖点 | 找冲突 | 💡 |
| 品牌语气 | 定调性 | 🎨 |
| 场景/受众 | 选舞台 | 🎬 |
| 竞品/风格 | 找差异 | 🔍 |

## 五、五大体验增强

### 5.1 右侧实时预览面板

纯前端——用户每填一张卡片，右栏实时更新 AI 理解的结构化摘要。底部微型进度条。

### 5.2 完成态卡片折叠

填完的卡片自动折叠为一行摘要 `✅ 产品名称: M5 Wearable Breast Pump [点击编辑]`。

### 5.3 步骤隐喻名 + Emoji

🎯定主角 → 💡找冲突 → 🎨定调性 → 🎬选舞台 → 🔍找差异

### 5.4 快捷模板

复用上次配置 / 品牌预设 / 从空白开始。

### 5.5 全部必填完成时微庆祝

底部按钮 `scale(1.02)` 脉冲 0.3s。

### 5.6 通用素材输入卡

所有场景（快速模式除外）末尾添加 🟢 可选素材卡——支持图片/视频/音频拖拽上传。上传后自动归档品牌资产。

**与品牌资产的双向循环**：创作引用库中素材 → 创作产出新素材 → 自动归入品牌资产。

## 六、执行页 & 结果页优化

### 6.1 进度叙事化

当前技术标签 → 大白话叙事文案。`S1_STEPS` 加 `narrative` 字段。

### 6.2 双模式执行状态条

| 模式 | UI | 
|------|-----|
| Smart Create | 底部固定胶囊条 + 取消按钮，页面可浏览 |
| Expert Studio | 保持 loading overlay（不改） |

新增 `isGenerating` 状态分离。

### 6.3 导演回放结果页

单页连续滚动：视频播放器 → 脚本摘要 → 关键帧画廊 → 质量报告 → 发布区 → 数据复盘 → 下载按钮。

### 6.4 创作小结卡片

右下角品牌色推送（3s 收起）。纯前端从 `result.audit_report` + `result.scripts` 提取。

### 6.5 发布区

位于结果页底部，无卡片边框，极简对话风格。AI 根据场景推荐平台，从策略数据自动填充标题/描述/标签。一键发布。

### 6.6 数据复盘看板

**按视频类型自适应北极星指标**：品牌片看声量，产品种草看 ROI，VLOG 看连接深度。

看板设计：AI 总结 → 指标比较（不用绝对值）→ 下一步建议→ 详细数据折叠区。

ROI 分解树仅销售型视频显示。

### 6.7 创作画廊

双 tab——成品(按场景分组 3 列卡片) + 素材(按类型分组)。TTS 音频不出现在画廊。空状态使用教练语言。

**导航调整**：原 `作品集/Footage` 改为 `创作画廊`。品牌资产的管理 CRUD 保留，素材展示迁移到画廊素材区。

### 6.8 品牌资产

品牌的内容操作系统——左侧导航树 + 右侧卡片网格。8 种资产类型各自不同卡片形态。

**自动归类**：创作流程中上传/生成的内容自动归入对应分类。跨类型搜索 + 来源筛选器。

## 七、现有架构约束 & 执行注意事项

> 此节基于对当前代码的最终审查（2026-04-30），确保计划不脱离实际。

### 7.1 文件规模风险

| 文件 | 当前行数 | v2.0 后预估 | 风险 |
|------|---------|-----------|------|
| `SceneForm.tsx` | 895 | 1000+ | 追加 GuidedForm 条件渲染将超限 |
| `page.tsx` | 1121 | 1200+ | 已极大，追加 isGenerating/底部条将进一步膨胀 |
| `OneShotResultView.tsx` | 832 | 1100+ | 追加发布区+数据复盘+画廊链接 |

**缓解**：Phase 2 执行时，将 GuidedForm 的逻辑尽量自包含在新建组件内，SceneForm 只做条件路由 `{scene !== "fast_mode" && <GuidedForm />}`。page.tsx 的 isGenerating 和底部条抽取为独立 hook `useExecutionBar`。

### 7.2 导航结构差距

当前 Nav 仅 2 项：首页、品牌资产。`fast_mode` 不在 SceneForm 中渲染——它通过 `FastModePanel` 直接在 page.tsx 第 844 行渲染。`footage/作品集` 通过 header 的 AssetLibrary 弹窗访问，不经过 Nav。

v2.0 需要的 Nav 结构：首页 / 创作画廊 / 品牌资产。Phase 8 需同时更新 Nav 组件。

### 7.3 FastMode 特殊处理

快速模式绕过 SceneForm，由 `page.tsx:844` 直接渲染 `FastModePanel`。v2.0 保留此行为——快速模式不使用 GuidedForm，不走卡片引导流。

### 7.4 数据流完整性

当前输入→执行→结果的数据流已通过 `content_scenario` 贯穿。发布阶段对 `briefs[0].topic` 和 `scripts[0].hashtags` 的引用需要验证——这些字段在 S1/S3/S5 的结果中始终存在，但在 S4（实拍素材）中可能缺失。缺失时使用空字符串作为 fallback。

## 八、全链路叙事闭环

```
🎯 定主角 → 💡 找冲突 → 🎨 定调性 → 🎬 选舞台 → 🔍 找差异
     ↓
🔄 正在分析产品 → 构思故事线 → 生成视频画面 → 拼接最终成片
     ↓
🎬 视频已完成 → 📝 创作脚本 → 🖼 关键帧 → 📊 质量报告
     ↓
🚀 准备好分享了吗 → AI 推荐平台 → 一键发布 → ✅ 已上线
     ↓
📊 完播率 72% → 比同类高 15% → 分享率偏低 → 优化 CTA
```

六段叙事：输入教练 → 执行叙事 → 导演回放 → 极简发布 → 数据复盘 → 创作画廊。信息自动衔接，零重复填写。
