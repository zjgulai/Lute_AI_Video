# 会说话的 UI — 矩阵式引导型输入框架设计

**日期**: 2026-04-30  
**灵感来源**: rabbithole.chat 设计理念 × 用户痛点 × Momcozy 品牌调性  
**目标**: 从「填表式」→「引导式」，让 UI 成为一个会说话的视频创作教练

---

## 一、当前问题 vs 设计目标

| 现状痛点 | 设计目标 |
|---------|---------|
| 密密麻麻表单，不同场景字段混杂在一起 | 矩阵导航：横轴选场景，纵轴选视频类型，卡片层展开 |
| 无数红色线条内容框，视觉压迫感重 | 品牌调性色标：栗红(必填) + 豆沙粉(建议) + 冷绿(可选) |
| 没有引导力，用户不知道为什么填 | 每张卡片三行：引导问题 → 原因说明 → 输入框 |
| 没有方法论框架 | 卡片间连接线 + 文字说明展示上下游输入关系 |

## 三、单张卡片规范

### 3.1 卡片结构（固定四行）

| 行 | 内容 | 样式 |
|----|------|------|
| 标签行 | 优先级标识 · 步骤编号 | 🔴/🟡/🟢 色标圆点 + 标签文字 · 12px |
| 问题行 | 引导问题（用户语言） | 15px · 品牌深色 · 半粗体 |
| 说明行 | 回答为什么 + 与视频的关系 | 12px · 品牌辅灰 · 行高 1.6 |
| 输入行 | 输入框/文本区/选择器 | 品牌输入框样式 |

### 3.2 卡片优先级视觉系统

| 层级 | 左边框颜色 | 标签文字 | 默认是否展开 |
|------|----------|---------|------------|
| 🔴 必填 | `#6A2B3A`  | "必填 · 第 N 步" | 始终展开 |
| 🟡 建议填 | `#B27A7E` | "推荐填写 · 第 N 步" | 始终展开 |
| 🟢 可选 | `#6B8578` | "可选项 · 第 N 步" | 默认折叠，点击展开 |

### 3.3 连接线设计

```
│ (品牌栗红细竖线, 2px)
│
├─ 引导文字 (12px, 品牌辅灰, 斜体, 左侧缩进)
│
▼ (箭头指向下一张卡片)
```

## 五、五大体验增强

### 5.1 右侧实时预览面板 (Live Summary Sidebar)

用户每填一张卡片，右栏自动更新。不调用 API——纯前端根据已填内容生成结构化摘要。

```
左 (卡片区)                    右 (AI 理解预览)
┌──────────────────┐          ┌────────────────┐
│ 🎯 定主角         │          │ AI 已理解:      │
│ 产品名称  ✓       │          │                │
│ [已折叠]          │          │ 主角: M5吸奶器  │
│                   │          │ 品类: 母婴      │
│ 💡 找冲突         │          │ 场景: 家庭/职场 │
│ 核心卖点  ▁▁▁     │          │ 情绪: 待补充    │
│ [正在填写...]     │          │                │
│                   │          │ 完成度: 40%     │
└──────────────────┘          └────────────────┘
```

不额外调用 LLM——纯前端提取逻辑：从产品名称推断品类、从卖点推断场景、从品牌语气提取情绪。右侧面板底部一个微型进度条。

### 5.2 完成态卡片折叠

填完的卡片离开焦点后自动折叠为一行摘要：

```
展开态:  折叠态:
┌────┐   ✅ 产品名称: M5 Wearable Breast Pump  [点击编辑]
│... │
└────┘
```

点击折叠态可重新展开编辑。始终只有当前卡片 + 上一张折叠卡片可见，保持视口清爽。

### 5.3 步骤隐喻名 + Emoji 图标

| 真实字段 | 步骤名 | 图标 |
|---------|--------|------|
| 产品名称 | 定主角 | 🎯 |
| 核心卖点 | 找冲突 | 💡 |

### 5.4 快捷模板

右上角「📋 模板」按钮，点击展开 4 个选项：

```

当所有 🔴 必填卡片状态为已完成时，底部「生成视频」按钮触发一次 `scale(1.02)` + 亮度微增的 0.3s 动画。不是弹窗——只是一个微妙的「准备好了」信号。

```

### 5.6 通用素材输入卡：每个场景的创作参考

**现状**：S1/S2/S5 无任何素材输入，S3 仅视频 URL，S4 仅实拍素材。Happy Horse 后端已支持 `image_to_video` 和 `reference_image_urls`，前端未暴露。

**修复**：所有场景（快速模式除外）在卡片序列最末尾添加一张通用 🟢 可选卡：

```
🟢 可选 · 有没有可以参考的素材？
上传参考图片、视频或音频，AI 会参考这些素材的风格生成视频。

🖼 参考图片               🎞 视频片段              🔊 音频
 (风格/产品图/竞品截帧)     (风格参考/竞品视频)       (品牌 jingle/语气参考)
 [点击上传]                [点击上传]               [点击上传]
```

| 场景 | 素材卡的额外说明 |
|------|--------------|
| S1 商品直拍 | 可以上传竞品视频作风格参考，或上传产品白底图 |
| S2 品牌宣传片 | 可以上传往期品牌片作调性参考 |
| S3 网红二创 | 已有 `video_url`，素材卡作为补充风格参考 |
| S4 实拍素材 | 已有 `footage_assets`，素材卡作为音频/补充参考 |
| **S5 品牌VLOG** | **六视图需要你批量上传 6 张产品角度图**，素材卡收集风格参考 |

**数据流**：上传的素材通过 `image_refs` 或 `first_frame_image` 参数传入 seedance 调用。同时按 6.8 的自动归类规则写入品牌资产——素材成为可复用的品牌内容。

**与品牌资产的循环**：品牌资产库中的素材 → 创作时引用 → 创作上传新素材 → 自动归档品牌资产。单向消费变成双向循环。

## 六、执行页 & 结果页优化

### 6.1 进度叙事化

当前进度条显示英文字段名 (`strategy ✓ → scripts ⏳ → compliance`)，改成大白话叙事：

| 步骤 | 当前 | 叙事文案 |
|------|------|---------|
| strategy | "策略" | 正在分析你的产品，理解核心卖点... |
| scripts | "脚本" | 已理解产品，正在构思故事线... |
| storyboards | "分镜" | 正在将脚本转化为镜头语言... |
| video_prompts | "视频提示词" | 为每个镜头编写 AI 视频生成指令... |
| seedance_clips | "AI 视频片段" | AI 正在生成视频画面，这一步需要 2-5 分钟... |
| tts_audio | "配音音频" | 正在用 AI 为脚本配音... |
| assemble_final | "合成最终成片" | 所有素材就位，正在拼接最终成片... |
| audit | "质量审计" | 进行最后的品质检查... |

实现：`S1_STEPS` 数组加 `narrative` 字段，`StageProgress` 渲染优先展示叙事文案。

### 6.2 双模式执行状态条

| 模式 | 执行中 UI | 用户行为 |
|------|---------|---------|
| Smart Create | 底部固定胶囊条 + 取消按钮，页面可正常浏览 | 浏览品牌资产、翻阅旧作 |
| Expert Studio | 保持 loading overlay + 取消按钮（不改） | 等待当前步骤完成 |

新增 `isGenerating` 状态与 `loading` 分离：`isGenerating` 用于非阻塞通知，`loading` 用于阻塞等待。

### 6.3 导演回放结果页

替代 `OneShotResultView` 的 4-tab 结构，改为单页连续滚动叙事：视频播放器 → 创作脚本摘要 → AI 关键帧画廊 → 质量报告 → 下载/重新生成按钮。

### 6.4 创作小结

生成完成时右下角推送品牌色小结卡片（3s 收起）：产品名、核心卖点出现次数、情绪基调、时长、质量评分、改进建议。纯前端从 `result.audit_report` + `result.scripts` 提取，无需后端改动。

### 6.5 发布阶段：从「技术控制台」到「分享你的作品」

**当前问题**：发布是一个独立的 DistributionView 页面，与前面的创作体验割裂。用户看完视频后被丢进一个 checkbox 表单页面，叙事弧线断裂。

**设计原则**：发布是创作的最后一步，不是独立工具。它应该接住用户刚刚产生的情感——「我创造了一个东西，现在我要让它被看到」。

**信息衔接链**：

```
输入页                执行页               结果页              发布区
GuidedForm  ──────→  叙事进度  ──────→  导演回放  ──────→  分享面板
                                                    (同页底部)
「你的产品是 M5」     「正在为 M5 生成」   「M5 视频已完成」   「把 M5 视频发到哪？」
```

**发布区位于导演回放结果页底部**——不跳页面，质量报告下方自然滚动进入。无卡片边框，无多余标签，像对话的延续：

```
... (质量报告内容向上滚动)

────────────────────────────────────────

  准备好分享了吗？

  因为这是品牌形象片，我们选了
  TikTok · Shopify 给你。

  ──────────────────────────────────

  M5 免手扶吸奶器
  为妈妈的舒适不断进化

  每天三秒佩戴，解放双手。职场妈妈的
  通勤神器。给忙碌妈妈多一点自由。
  #免手扶吸奶器 #Momcozy #职场妈妈

  ──────────────────────────────────

  [ 发布到 TikTok · Shopify ]
```

**极简原则**：

- 平台不展示为卡片，而是自然语句中嵌入的名称
- 标题和描述不显示 input 框——像已经排版好的最终文案，点击即可编辑
- 只有一个按钮，按钮文字直接说明发去哪
- 没有「标签」「描述」「标题」等字段标签——视觉上像成品的预览，不是表单
- AI 自动推荐平台直接在文字中用 `·` 分隔，用户点击名称可取消该平台

**平台选择交互**：默认 AI 推荐的平台用实色，未被推荐的平台用淡色。点击切换选中态：

```
TikTok  ·  Shopify  ·  YouTube  ·  Amazon

(品牌形象片的 AI 推荐是 TikTok + Shopify，这两个用实色，其余淡色)
```

**发布完成**：按钮替换为一行静默确认文字，不做弹窗：

```
✅ 已发布到 TikTok · Shopify
   TikTok 预计 2 小时内完成审核
   Shopify 已上线产品详情页
```

**信息衔接**（数据从前面阶段自动流入，用户无需重复填写）：

| 发布字段 | 数据来源 | 
|---------|---------|
| 推荐平台 | 输入阶段 `content_scenario`（品牌宣传片→TikTok+Shopify+YouTube；网红二创→Shopify+Amazon+TikTok） |
| 标题 | `briefs[0].topic` + 品牌名拼接 |
| 描述 | `scripts[0].segments` 前 3 段 voiceover，截取 150 字 |
| 标签 | `scripts[0].hashtags` 直接复用 |

**全链路叙事闭环**：

```
🎯 定主角  →  💡 找冲突  →  🎨 定调性  →  🎬 选舞台  →  🔍 找差异
     ↓
🔄 正在分析你的产品 → 正在构思故事线 → 正在生成视频画面 → 拼接最终成片
     ↓
🎬 你的视频已完成 → 📝 创作脚本 → 🖼 AI 关键帧 → 📊 质量报告
     ↓
🚀 准备好分享了吗？ → AI 建议 TikTok + Shopify → 一键发布 → ✅ 已上线
     ↓
📊 完播率 72% → 比同类高 15% → 分享率偏低 → 下条优化结尾 CTA
```

**完整六段叙事弧线**：输入教练 → 执行叙事 → 导演回放 → 极简发布 → 数据复盘。用户从输入到复盘始终在同一种教练语言中，信息全程自动衔接，零重复填写。

### 6.7 创作画廊：成品与素材的双轴浏览

**命名**：`作品集/Footage` → `创作画廊`。不是文件仓库，是视频创作成果的展览。

**双轴结构**：顶部 tab —「成品」「素材」。默认「成品」。切换时分组淡入淡出。

| 成品（按场景分组） | 素材（按类型分组） |
|------------------|------------------|
| 商品直拍 (3) | 实拍片段 (12) |
| 品牌VLOG (1) | 品牌资产 (5) |
| 网红二创 (2) | 音频素材 (3) |
| 品牌宣传片 (1) | |

**卡片规格**：`max-w-5xl` (1024px) 内 3 列网格，每卡 3:2 比例，含场景图标、标题、一个关键指标、时间戳。

**音频展示规则**：TTS 生成配音不出现在画廊——仅在视频详情页展示为「制作素材」。独立上传的品牌音频（jingle、音效）出现在素材区。

**空状态**：教练语言 —「你还没有生成视频，回到首页开始你的第一个创作吧。」

**全链路衔接**：导演回放结果页底部链接 `→ 查看创作画廊`，跳转后高亮该视频卡片。

**与品牌资产页关系**：品牌资产的管理 CRUD 保留在 `brand-packages` 页中，但「展示」内容迁移到画廊的素材区，用户在一处浏览所有创作相关物料。

### 6.8 品牌资产：品牌的内容操作系统

**重新定义**：品牌资产不只是配置表单——是品牌的**身份 + 全部产出**的统一仓库。

```
品牌资产

┌─ 品牌身份 ──────────────┐  ┌─ 已产出内容 ─────────────┐
│                          │  │                          │
│ 📋 品牌 Brief             │  │ 🎬 AI 视频 (按场景来源)  │
│ 👥 用户画像               │  │ 🖼 AI 图片 (关键帧/缩略图) │
│ 🎨 视觉规范 (色板/Logo/字) │  │ 🔊 AI 音频 (TTS/jingle)  │
│ 📦 产品图片 (SKU六视图)    │  │ 📖 品牌故事视频           │
│ 🎞 参考素材 (竞品/风格)    │  │ ✨ 品牌原创 (非AI产出)    │
└──────────────────────────┘  └──────────────────────────┘
```

**自动归类规则**：用户在创作流程中上传或生成的内容，自动进入品牌资产的对应分类：

| 来源场景 | 产生内容 | 自动归入 |
|---------|---------|---------|
| S5 品牌VLOG 上传产品图 | SKU 六视图 | 品牌身份 → 产品图片 |
| S3 网红二创上传视频 | 原始口播素材 | 已产出 → 品牌原创 |
| S1/S5 生成完成 | 最终 MP4 + 缩略图 | 已产出 → AI 视频 + AI 图片 |
| TTS 步骤完成 | 配音 mp3 | 已产出 → AI 音频 |
| S4 上传实拍片段 | 实拍素材 | 已产出 → 品牌原创 |

**交互**：左侧垂直导航树按资产类型分组，右侧卡片网格。不同资产类型不同卡片形态——视频用 3:2 横向卡，图片用正方形缩略网格，音频用波形条卡片，文本类用摘要卡。顶部跨类型搜索 + 来源筛选（AI生成/人工上传/品牌导入）。

**与创作画廊关系**：画廊聚焦「成品 + 复盘」，品牌资产聚焦「原材料 + 全品类」。视频在两边共享数据不重复存储——画廊的卡片打开是完整报告，品牌资产的卡片打开是素材视角。

### 6.6 数据复盘看板：按视频类型的成绩单

**核心理念**：不同视频类型的「成功」定义不同——输入阶段的选择决定了复盘阶段的北极星指标。

| 视频类型 | 北极星指标 | 核心漏斗 | 代表问题 |
|---------|----------|---------|---------|
| 产品解说/种草 | 成交 ROI | 曝光→点击→加购→下单 | 「这条视频卖了多少？」 |
| 品牌形象片 | 品牌声量 | 完播率→分享率→评论情感→粉丝增长 | 「观众记住了吗？有好感吗？」 |
| 开箱测评 | 信任转化 | 完播率→详情页点击→加购率 | 「观众看完更想买了吗？」 |
| 网红二创 | 带货转化 | 曝光→链接点击→下单→分佣 | 「达人带来了多少单？」 |
| 品牌VLOG | 连接深度 | 完播率→互动率→粉丝增长 | 「观众对品牌更有感情了吗？」 |
| 活动宣发 | 参与转化 | 曝光→点击→活动页→报名 | 「活动带来了多少参与者？」 |

**看板设计：先总结，再数据，后建议**

```
（导演回放页底部发布成功后，自然过渡到数据区）

📊 你的 M5 品牌形象片 发布数据

这条视频完播率 72%，比你的其他品牌形象片高 15%——
但分享率只有 2%。也许结尾的"舒服吗？"钩子还不够打动观众。

  ─────────────────────────────────

  完播率 72%          分享率 2%          粉丝增长 +48
  好↑15%  ▁▄▇        ＜平均  ▁▂        TikTok 最多

  ─────────────────────────────────
  下一步建议

  · 试试在结尾加一句"转发给正在泵奶的妈妈"
  · 完播率说明你的中段节奏很好，保持这个脚本结构
  · 下一条试试缩短前 3 秒，抢夺视频流的注意力

  📈 查看详细数据     🎬 基于这条再创作
```

**设计原则**：
① 顶部 AI 总结——纯前端从指标数据中提取 max/min，拼接成一句话结论
② 指标用比较（↑15%），不用绝对值（3.2%）
③ 每条指标下方跟一句白话解释 + 一条行动建议
④ 无筛选器、无标签页、无数据表格——用户不需要操作，直接阅读
⑤ 「查看详细数据」折叠区放原始数据表，供深度分析用户展开

**ROI 分解树（仅产品种草/开箱测评/网红二创显示）**：

```
ROI 320%  ←── 销售额 $960 / 成本 $300
│
├── 曝光 12,400 次    TikTok 分发
├── 点击率 3.8%       封面钩子有效
├── 加购率 12%       视频说服力强  ← 你的优势
└── 下单率 68%       价格/详情页  ← 可优化空间

哪里需要提升？↓
  下单率低于同类均值（85%），试试在视频结尾增加价格锚点
```

**信息衔接**：看板不需要用户额外配置——视频类型从输入阶段携带，平台数据从发布阶段关联。用户从「导演回放」滚动到「发布区」再滚动到「数据复盘」，全程不需要导航跳转。
