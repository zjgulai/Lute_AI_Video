# 会说话的 UI 2.0 — Brainstorming 核心结论与方法论总结

**日期**: 2026-04-30  
**来源**: 历时 8 小时的全链路 Brainstorming 会话  
**定位**: UI 2.0 的指引路线文件，浓缩全部的核心理念、方法论和可复用模式

## 一、核心诊断：当前产品的 UI 本质问题

经过对 75+ 源文件的深度审查和与 rabbithole.chat 的对照分析，我们的产品 UI 存在三个结构性缺陷：

### 1.1 缺失「为什么」—— 用户不被告知填写的意义

当前场景表单是纯粹的数据收集器。用户面对密密麻麻的 input 框，不知道：
- 这个字段填了会对视频产生什么影响？
- 前一个字段和后一个字段有什么关系？
- 哪些字段是决定性的，哪些是锦上添花的？

**核心洞察**：表单不应该只问「填什么」，应该同时回答「为什么要填」和「填了会怎样」。

### 1.2 缺失「方法论」—— 用户看不到创作框架

当前产品没有向用户展示视频创作的内在逻辑。用户不知道「产品名称 → 核心卖点 → 品牌语气 → 目标受众 → 竞品差异」是一个有因果关系的创作链。

**核心洞察**：工具型产品最大的用户价值不是「帮你做」，而是「让你懂」。当一个用户完成三次创作后，他应该自己学会了如何构思一条视频。

### 1.3 缺失「连续性」—— 从输入到结果到发布是割裂的

当前全链路：输入页（SceneForm）→ 执行页（StageProgress/loading overlay）→ 结果页（OneShotResultView tab 切换）→ 发布页（DistributionView 独立页面）——每一个阶段都是视觉和交互上的硬切换，用户每次切换都会丢失上下文。

**核心洞察**：用户心智中只有一个进程——「我在创作一条视频」。UI 应该反映这个心智模型，而不是把它拆成四个独立页面。

## 二、会说话的 UI —— 核心方法论

### 2.1 教练语言原则

> UI 不应该是一个工具箱。它应该是一个老师，一步步带着用户做完一件事。

**具体表现**：

| 维度 | 工具箱语言（当前） | 教练语言（v2.0） |
|------|------------------|-----------------|
| 字段标签 | "产品名称" | 🎯 定主角 · 你的视频主角是谁？ |
| 错误提示 | "请输入产品名称" | 不阻挡，不催促——缺少必填信息时「生成」按钮保持禁用，但不弹错误 |
| 进度展示 | "strategy ✓ → scripts ⏳" | "正在分析你的产品，理解核心卖点..." |
| 结果展示 | 4 个 Tab 页 | 导演回放：视频 → 脚本 → 画廊 → 报告，自然滚动 |
| 空状态 | "暂无数据" | "你还没有生成视频，回到首页开始你的第一个创作吧" |

### 2.2 信息连续性原则

> 同一份信息，用户在链路中只输入一次。后续所有节点自动继承。

**具体实现**：

```
输入阶段填了 content_scenario
  → 发布阶段自动推荐匹配平台
  → 数据复盘阶段自动选择北极星指标

输入阶段填了 product_name
  → 执行阶段进度条显示"正在为 M5 生成..."
  → 结果页标题自动填充
  → 创作小结自动提取产品名

输入阶段填了 core_message
  → 发布区描述自动从脚本前 3 段提取
  → 数据复盘总结句自动包含核心卖点
```

### 2.3 渐进降低认知负荷原则

> 用户不应该一次性面对所有输入项。应该按优先级和因果关系逐步展开。

**三级优先级系统**：

| 层级 | 名称 | 颜色 | 含义 | 默认状态 |
|------|------|------|------|---------|
| 🔴 | 必填 | `#6A2B3A` | 没有这个视频无法生成 | 展开 |
| 🟡 | 建议填 | `#B27A7E` | 填了质量显著提升 | 展开 |
| 🟢 | 可选 | `#6B8578` | 锦上添花 | 折叠 |

连接线解释上下游关系：「产品定了，视频就有了主角」 → 「卖点是论据，每个镜头都在证明它」 → 「价值观决定镜头情绪温度」

### 2.4 按视频类型差异化原则

> 不是所有视频的成功标准都一样。品牌片和种草片的北极星完全不同。

| 视频类型 | 北极星指标 | 成功定义 |
|---------|----------|---------|
| 产品种草/解说 | 成交 ROI | 这条视频卖了多少？ |
| 品牌形象片 | 品牌声量 | 观众记住了吗？有好感吗？ |
| 开箱测评 | 信任转化 | 观众看完更想买了吗？ |
| 网红二创 | 带货转化 | 达人带来了多少单？ |
| 品牌VLOG | 连接深度 | 观众对品牌更有感情了吗？ |
| 活动宣发 | 参与转化 | 活动带来了多少参与者？ |

这个差异化从输入阶段就开始——选场景 → 选视频类型 → 自动匹配卡片序列 → 自动匹配北极星 → 自动匹配数据复盘展示。

## 三、用户价值

### 3.1 降低使用门槛

「工具箱」语言要求用户自己知道哪些字段重要、按什么顺序填、每项填什么内容。首次使用用户需要学习 10-15 分钟。

「教练」语言在每一步都告诉用户「为什么填」和「填完会怎样」。首次使用用户 2-3 分钟可以完成第一次创作。

### 3.2 提升创作质量

引导卡片不只是收集信息——它在教用户方法论。当用户填完 🎯定主角 → 💡找冲突 → 🎨定调性 → 🎬选舞台 → 🔍找差异 五步后，他实际上完成了一次视频策略的框架性思考。填得越细，视频质量越高。

### 3.3 建立用户忠诚

每次使用都是一次「被引导的良好体验」，而不是「被丢进一个工具自己摸索」。用户在完成 3-5 次创作后，自然学会了这套方法论，可以在没有 AI 的情况下自己规划视频内容。这是工具的最高境界——不是让你依赖它，而是让你通过它成长。

## 四、可复用的设计模式

### 4.1 引导卡片模式 (Guided Card Pattern)

**适用场景**：任何需要用户按顺序输入结构化信息的 AI 工具。

**核心结构**：
```
优先级标识 + 步骤编号
引导问题（大白话，用户语言）
原因说明（为什么填 + 填了会怎样）
输入区域
```

**可复用要素**：
- 三级优先级色标系统（必填/建议/可选）
- 步骤隐喻名 + Emoji（定主角🎯/找冲突💡/定调性🎨/选舞台🎬/找差异🔍）
- 连接线 + 引导文字展示上下游关系
- 填完自动折叠，保持视口清爽

### 4.2 实时预览模式 (Live Preview Pattern)

**适用场景**：AI 工具在用户填写过程中需要给出「我理解了什么」的反馈。

**核心机制**：不调用 LLM，纯前端从已填内容提取结构化摘要。右栏展示 → 完成度进度条 → 全部完成后底部按钮微庆祝。

**可复用要素**：
- 每次字段变更触发摘要更新
- 摘要用自然语言而非 JSON
- 微型进度条暗示「快完成了」

### 4.3 滚动叙事结果页 (Director's Playback Pattern)

**适用场景**：AI 生成完成后需要展示复杂产出物（视频、脚本、图片、数据）。

**核心机制**：替代 Tab 切换的单页滚动叙事——视频播放 → 脚本摘要 → 图片画廊 → 质量报告 → 发布区 → 数据复盘 → 下载。

**可复用要素**：
- 信息按用户查看优先级排序（视频第一，数据最后）
- 每段之间用细横线隔开
- 自然向下滚动，不需要导航跳转

### 4.4 双轴浏览模式 (Dual-Axis Gallery Pattern)

**适用场景**：用户回看历史产出——需要按不同维度浏览。

**核心机制**：顶部 Tab 切换浏览轴——「成品」（按场景分组）「素材」（按类型分组）。卡片 3 列网格，3:2 比例，浓缩关键指标。

**可复用要素**：
- Tab 切换分组而不是筛选器下拉
- 每张卡片只展示一个关键指标（而非全部字段）
- 空状态用教练语言引导回首页

### 4.5 全品类资产仓库模式 (Brand OS Pattern)

**适用场景**：品牌的所有创造性产出需要一个统一入口。

**核心机制**：左导航树（品牌身份/已产出内容）+ 右卡片网格（自适应不同资产类型的卡片形态）。

**可复用要素**：
- 不按文件类型分组（视频/图片/音频），按「来源」分组（AI生成/人工上传/品牌导入）
- 自动归类：创作流程中产生的内容无需用户手动归位
- 与创作形成双向循环：库中素材→创作引用→新产出→自动归档

### 4.6 差异化北极星模式 (Multi-North-Star Pattern)

**适用场景**：同一平台服务于不同类型的任务，每类的成功标准不同。

**核心机制**：任务类型决定评估体系——不需要用户手动选择看哪个指标。

**可复用要素**：
- 从输入阶段的选择自动推导复盘阶段的北极星
- 指标用比较不用绝对值（↑15% 而非 3.2%）
- 每条指标下方跟一条白话解释 + 一条行动建议

## 五、方法论适用性判断

| 如果你的产品有以下特征 | 我们的方法论适用度 |
|---------------------|-----------------|
| AI 辅助创作类工具 | ⭐⭐⭐⭐⭐ 完全适用 |
| 需要用户输入结构化信息的 SaaS | ⭐⭐⭐⭐ 高度适用 |
| 多步骤操作类产品 | ⭐⭐⭐⭐ 高度适用 |
| 数据看板/BI 工具 | ⭐⭐⭐ 部分适用（差异化北极星、白话总结） |
| 社交/内容消费类产品 | ⭐⭐ 低级适用 |
| 纯工具/编辑器类产品 | ⭐⭐ 低级适用 |
