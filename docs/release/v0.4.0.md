## TL;DR

Sprint 0-3 cumulative release covering Phase 0 mandatory fixes + 13 follow-up improvements. ## 一、Features

### Phase 0 — Production-readiness fixes

- **PG runtime state persistence** (`7a2f4b8c9d12`): 5 columns added to `pipeline_states` (`schema_version`, `pipeline_degraded`, `degraded_reason`, `trace_id`, `structured_errors`). Fixes Oracle-identified regression where degraded state was lost on PG round-trip.
- **BudgetExceededError → degraded chain**: Expert mode budget guard now flows through `step_runner`'s `try` block instead of bubbling as HTTP 500.
- **Severity-aware medical lexicon**: BrandCompliance no longer hard-blocks benign FLAGGED terms (e.g. "natural lighting", "organic cotton"). 244-term lexicon split into BANNED / FLAGGED / COMPETITOR with per-term severity.

### Phase 2 — Observability

- **5 new Prometheus metrics** (commit `4fc6428`):
  - `llm_api_errors_total{provider, error_kind}` Counter
  - `llm_api_duration_seconds{provider}` Histogram
  - `db_pool_available_connections` + `db_pool_size` Gauges
  - `admin_login_attempts_total{outcome}` Counter
  - `tenant_active_count` Gauge
- **Docker log rotation**: 100MB max-size × 3-file rotation on all 4 services. Prevents disk-fill deadlock.
- **Phase 0 watchdog cron**: Hourly SLI check (empty final_video / heuristic scoring / stuck runs). Logs to `/var/log/phase0_watchdog.log`.
- **audit_logs table** (`9f1e2c8a4b67`): Business-event audit trail with 11-column schema + 4 indexes. Admin login/logout events now emit. Other events (tenant CRUD, pipeline lifecycle, api_key revoke) wired in follow-ups.

### Security

- **API key log leak fix**: `_deps.py` no longer prints temporary API_KEY value to startup logs.
- **`.env.prod` chmod 600**: Production secrets file now restricted to owner-read only.
- **Admin CSRF double-submit token**: All admin POST/PUT/DELETE endpoints require `X-CSRF-Token` header matching `admin_csrf` cookie. Login response sets cookie + returns token in body.
- **Removed CORS dead config**: `https://*.tcloudbaseapp.com` wildcard (never worked, Starlette doesn't expand subdomain wildcards) removed.

### S2-S5 enhancements

- **S2 Brand Campaign v2** (independent pipeline): `kling-3.0/pro` 1080p as primary, 4-clip multi-shot, deprecation shim from old `s2_brand_pipeline.py` preserves backward compat.
- **S5 Brand VLOG** multi-segment: `seedance-2` 15s × N segments with `VideoContinuityManager` last-frame anchoring. Unlocks 30/45/60/90s VLOG durations.
- **S3/S4/Fast Mode** also through `ModelRouter`: scenario-aware model selection chains.
- **Remotion multi-aspect-ratio fan-out**: 9:16 + 1:1 + 16:9 from same source.

### Gate / Quality

- **Model-aware thresholds**: 25+ POYO/Seedance/Kling models × 7 threshold knobs. Replaces global 0.65 cutoff.
- **`gpt-4o` vision scoring**: Keyframe evaluation now reads image content, not just prompt keywords.
- **Scenario-aware weights**: S1-S5 each have 6-dimension scoring weights tailored to their judgment criteria.
- **Quality feedback_gate skeleton**: `evaluate_upstream_quality()` returns 3-state decision (proceed/warn/regenerate). Pilot keyframe consumer integration in follow-up.

### Resilience

- **partial_artifacts stub-aware**: Detects all-stub seedance_clips (mock-mode artifact) and classifies as missing, not available. Closes silent assemble failure.
- **C2PA signing scaffold**: Env-gated (`C2PA_ENABLED=1`), graceful degradation if cert missing. Activates before EU AI Act 2026-08-02.
- **Schema versioning + load-time mismatch warning**: Forward/backward compat marker per pipeline state record.

### Frontend

- **/sN?label=X&gate=Y direct gate route**: 4 scenario pages now support direct-link access to Gate review without going through full workflow.
- **i18n**: 24 SCENE_VIDEO_TYPES descriptions translated. `brand_vlog` scenario title/desc consistency fix.
- **`catch (e: any)` → `catch (e: unknown)` + `errorMessage()` helper**: 41 sites, -63 `no-explicit-any` eslint errors.
- **nursery scene removed** (baby-safety compliance): aligns with backend `_validate_s5_scene_id` reject list.

### DX / CI

- **GitHub Actions enriched**: 5 jobs total \u2014 lint / test / docs-link-check (lychee) / frontend-test (vitest + tsc) / docker-build (validate, no push).
- **CONTRIBUTING.md**: 30-min new-developer onboarding guide.
- **Phase 0 deploy SOP** + Aliyun apt mirror in Dockerfile.backend: avoids 74-min apt hang from mainland China.
- **psycopg v3 alembic dialect**: `migrations/alembic/env.py` rewrites URL scheme to use v3 instead of missing psycopg2.

## 三、Behavior changes user-visible

### Admin panel users

- **MUST re-login after release**: existing admin sessions issued before this release lack the new `admin_csrf` cookie. Any state-changing action (create tenant, revoke key, etc.) returns 403 until re-login. 24h cookie expiry, max 1 day of friction. No data loss.
- Login response now includes `csrf_token` field. Frontend automatically picks it up via `document.cookie` parsing.

### API consumers (tenant API keys)

- **No changes** required. `X-API-Key` header unchanged.
- New observability: caller errors logged with `error_kind` label for Prometheus aggregation. No payload changes.

### Pipeline degradation behavior

- BudgetExceededError now returns HTTP 200 with `pipeline_degraded: true` + `degraded_reason: "budget_exceeded"`. Previously returned HTTP 500.
- Frontend already handles degraded state (commit `db89079`); UI shows partial artifacts.

### Content moderation

- "natural", "organic", "doctor recommended" and 110 similar marketing-safe terms no longer hard-block compliance. They surface as `severity: "low"` flags. Strictly medical claims (treat / cure / diagnose) still BLOCKED.
