# AI Video Pipeline — API 平台深度调研报告

> 调研日期：2026-04-29
> 调研目标：替代 poyo.ai，找到兼具稳定性、性价比、响应速度的 API 方案，结合 Kimi API 制定各节点模型分配计划

---

## 一、执行摘要

### 当前问题
- **poyo.ai 余额不足**（402 insufficient_credits_error），导致图像生成、视频生成、语音生成全部失败
- poyo.ai 采用积分制，余额耗尽时直接硬失败，无优雅降级
- 需要找到更稳定、更具性价比、响应更快的替代方案

### 核心结论
- **文本类任务**（策略、脚本、文案）：**Kimi K2**（官方 API）— 中文最强、速度最快、成本最低
- **图像生成**（关键帧、缩略图）：**硅基流动 Kolors/FLUX** — 国内服务器、¥0.3/张、1秒出图
- **视频生成**（片段）：**硅基流动 HunyuanVideo** — 国内服务器、开源模型、速度快
- **语音生成**（TTS/旁白）：**硅基流动 CosyVoice** — 国内服务器、100ms响应、部分免费
- **图像理解/审核**：**Kimi K2 多模态** — 已支持，无需额外配置

**推荐架构：Kimi（文本+理解）+ 硅基流动（图像+视频+语音）**

## 二、poyo.ai 深度分析

### 服务范围
| 模态 | 覆盖模型 | 定价 |
|---|---|---|
| 图像生成 | Nano Banana ($0.025/张)、Seedream 5.0、GPT-4o Image | 积分制，比官方低30-80% |
| 视频生成 | Sora 2 ($0.15/条)、Veo 3.1、Wan 系列 | 积分制 |
| 语音/音乐 | Suno v5、ElevenLabs | 积分制 |
| LLM | Claude Sonnet 4.5、Gemini 3、GPT-5 | 积分制 |

### 问题诊断
1. **余额管理困难**：积分制，无预警机制，耗尽直接 402
2. **无优雅降级**：所有媒体生成硬失败，pipeline 产出"空壳"视频
3. **代理层风险**：多一层代理，多一层故障点
4. **国内速度一般**：服务器在海外，国内访问延迟较高

## 四、Kimi API 能力评估

### 已验证能力 ✅
- **文本生成**：Kimi K2/K2.5，中文最强，支持超长上下文
- **代码生成**：Kimi Coding Plan，编程能力优秀
- **图像理解**：多模态输入，可分析图片内容
- **思考模式**：enable_thinking 参数，推理深度好

### 不支持 ❌
- **图像生成**：Kimi API 只能"理解"图片，不能"生成"图片
- **视频生成**：不支持
- **语音生成**：不支持

### 结论
Kimi 是**文本类任务**的最佳选择，但**多媒体生成**必须借助其他平台。

## 五、推荐方案：Kimi + 硅基流动

### 5.1 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                     AI Video Pipeline                        │
├─────────────────────────────────────────────────────────────┤
│  策略生成 (strategy)     │  Kimi K2 (官方)                   │
│  脚本生成 (script)       │  Kimi K2 (官方)                   │
│  文案/审核 (copy/audit)  │  Kimi K2 (官方)                   │
├─────────────────────────────────────────────────────────────┤
│  关键帧生成 (keyframe)   │  硅基流动 Kolors / FLUX           │
│  缩略图生成 (thumbnail)  │  硅基流动 Kolors / FLUX           │
├─────────────────────────────────────────────────────────────┤
│  视频片段 (seedance)     │  硅基流动 HunyuanVideo / Wan2.1   │
├─────────────────────────────────────────────────────────────┤
│  语音/旁白 (tts)         │  硅基流动 CosyVoice               │
├─────────────────────────────────────────────────────────────┤
│  图像理解 (审核/分析)    │  Kimi K2 多模态                   │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 各节点详细分配

| Pipeline 节点 | 当前使用 | 推荐替代 | 模型名称 | 原因 |
|---|---|---|---|---|
| `strategy_node` (策略) | poyo → GPT/Claude | **Kimi K2** | `kimi-k2-0905-preview` | 中文策略生成最强，推理深度好 |
| `scripts` (脚本) | poyo → GPT/Claude | **Kimi K2** | `kimi-k2-0905-preview` | 中文脚本自然流畅 |
| `keyframe_images` (关键帧) | poyo → GPT Image | **硅基流动** | `Kwai-Kolors/Kolors` 或 `black-forest-labs/FLUX.1-schnell` | ¥0.3/张，1秒出图，国内速度快 |
| `seedance_clips` (视频片段) | poyo → Seedance/Sora | **硅基流动** | `tencent/HunyuanVideo` 或 `Wan-AI/Wan2.1-T2V-14B` | 国内服务器，开源模型，性价比高 |
| `tts` (语音旁白) | poyo → ElevenLabs | **硅基流动** | `FunAudioLLM/CosyVoice2-0.5B` | 100ms响应，部分免费，中文TTS优秀 |
| `thumbnail_images` (缩略图) | poyo → GPT Image | **硅基流动** | `Kwai-Kolors/Kolors` | 同上 |
| `audit` (质量审核) | poyo → GPT/Claude | **Kimi K2** | `kimi-k2-0905-preview` | 多模态理解，可分析视频帧 |

### 5.3 成本估算（单条视频）

假设生成 1 条 15 秒产品视频：

| 环节 | 用量 | poyo.ai 成本 | 新方案成本 |
|---|---|---|---|
| 策略+脚本（文本） | ~5K tokens | 积分 | Kimi: ~¥0.01 |
| 关键帧图片 | 2-3 张 | ~$0.05-0.15 | 硅基: ~¥0.6-0.9 |
| 视频片段 | 1-2 段 | ~$0.15-0.30 | 硅基: ~¥1-2 |
| 语音旁白 | 1 段 | 积分 | 硅基: ~¥0.1 或免费 |
| 缩略图 | 2 张 | ~$0.05-0.10 | 硅基: ~¥0.6 |
| **合计** | — | **~$0.30-0.60** | **~¥2.5-4 (~$0.35-0.55)** |

> 成本基本持平，但**稳定性**和**响应速度**大幅提升。

1. 更新 `.env`：
   ```
   # 文本 LLM
   DEFAULT_LLM_PROVIDER=kimi
   KIMI_API_KEY=你的KimiKey
   
   # 多媒体生成
   SILICONFLOW_API_KEY=你的硅基流动Key
   
   # poyo 作为备选（留空则禁用）
   POYO_API_KEY=""
   ```

### Phase 4: 测试验证（2-4小时）
1. 本地测试各节点调用
2. 跑通完整 pipeline
3. 对比生成质量和速度

## 七、风险提示

1. **硅基流动视频模型质量**：HunyuanVideo/Wan 是开源模型，质量可能不如 Seedance/Sora，需实际测试
2. **接口适配成本**：硅基流动是 OpenAI-compatible 格式，与当前 poyo 的异步 submit+poll 架构不同，需要封装适配层
3. **双 Key 管理**：需要同时维护 Kimi Key 和硅基流动 Key
